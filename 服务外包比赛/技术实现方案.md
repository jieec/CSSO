# 故障树智能生成系统 - 技术实现方案

## 一、核心技术栈（全部开源）

### 1. AI模型层
- **开源大模型**：
  - Llama 3（Meta开源，70B/8B参数）
  - GLM-4（智谱AI开源）
  - ChatGLM3-6B（清华开源）
- **模型部署**：
  - 本地部署：Transformers + PyTorch
  - 云端API：阿里百炼、智谱AI开放平台
- **选型依据**：
  - 中文理解能力强
  - 支持长文本处理（设备手册通常较长）
  - 开源可商用

### 2. 知识提取技术
- **NLP技术**：
  - 命名实体识别（NER）：提取故障事件
  - 关系抽取（RE）：识别因果关系
  - 文本分类：区分顶/中/底事件
- **Prompt工程**：
  - Few-shot Learning：提供示例指导模型
  - Chain-of-Thought：引导逻辑推理
  - 结构化输出：JSON格式约束

### 3. 故障树构建算法
- **图算法**（NetworkX）：
  - DAG构建：有向无环图
  - 拓扑排序：确定层级
  - 路径分析：故障传导路径
- **布局算法**：
  - Graphviz DOT：层次布局
  - Force-directed：力导向布局

### 4. 逻辑校验
- **规则引擎**：
  - 循环检测：is_directed_acyclic_graph
  - 孤立节点检测：isolates
  - 逻辑门规则：输入数量、类型检查
- **AI辅助**：
  - 异常模式识别
  - 优化建议生成

def generate_fault_tree(events, gates, top_event):
    # Step 1: 构建图
    G = nx.DiGraph()
    add_nodes_and_edges(G, events, gates)
    
    # Step 2: 逻辑校验
    if not nx.is_directed_acyclic_graph(G):
        raise ValueError("存在循环依赖")
    
    # Step 3: 层级计算
    levels = compute_levels(G, top_event)
    
    # Step 4: 布局计算
    positions = compute_layout(G, levels)
    
    # Step 5: 生成输出
    return build_tree_data(G, positions)
```

def calculate_accuracy(generated_tree, ground_truth):

    structure_acc = compare_structure(
        generated_tree.nodes,

    )
    
    # 关联准确率

def preprocess_files(file_paths):
    all_text = []
    for path in file_paths:
        if path.endswith('.docx'):
            text = extract_from_docx(path)
        elif path.endswith('.xlsx'):
            text = extract_from_excel(path)
        elif path.endswith('.pdf'):
            text = extract_from_pdf(path)
        else:
            text = read_text_file(path)
        
        all_text.append(text)
    
    return merge_and_clean(all_text)
```
