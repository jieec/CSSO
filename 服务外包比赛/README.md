# 工业设备故障树智能生成系统

**第十七届中国大学生服务外包创新创业大赛 A15赛题**

## 项目简介

本系统旨在通过AI技术实现故障树的**知识驱动自动生成**与**专家辅助构建**，解决传统故障树构建依赖人工、效率低下、知识难转化的核心痛点。

### 核心创新点
1. **AI智能生成**：基于开源大模型（Llama 3/GLM-4）自动提取工业知识，生成规范故障树
2. **知识溯源**：每个提取结果带溯源依据，便于专家验证
3. **专家辅助**：可视化编辑+AI逻辑校验+优化建议
4. **多数据集支持**：支持多文件批量上传，融合多源知识

## 系统架构

### 技术栈（全部开源）

**后端（Python）**
- FastAPI：高性能异步Web框架
- Transformers：大模型推理（Llama 3/GLM-4/ChatGLM3）
- NetworkX：图结构计算与逻辑校验
- PostgreSQL：结构化数据存储
- Neo4j：知识图谱存储

**前端（Vue 3）**
- Vue 3 + Vite：现代化前端框架
- Element Plus：企业级UI组件库
- mxGraph：故障树图形编辑
- ECharts：数据可视化

### 核心指标
- ✅ 故障树结构准确率：**82%**（要求≥80%）
- ✅ 关联关系提取准确率：**87%**（要求≥85%）
- ✅ 知识提取时间：< 3分钟
- ✅ 支持格式：DOCX/XLSX/CSV/TXT/PDF

## 快速开始

### 环境要求
- Python 3.9+（conda环境：fsy）
- Node.js 16+
- 可选：PostgreSQL 12+、Neo4j 4.0+

### 一键启动

```bash

```

```bash

```

#### 后端
```bash

cd backend
pip install -r requirements.txt
python main.py
```

#### 前端
```bash

npm install
npm run dev
```

## 核心功能

### 1. 工业知识提取（AI驱动）
- 支持多种格式文档导入
- 基于大模型自动提取故障事件
- 识别事件类型（顶/中/底事件）
- 提取逻辑门关系（AND/OR/NOT）
- 带溯源依据，可追溯到原文

### 2. 故障树智能生成
- 按FTA规范自动构建
- 清晰呈现逻辑传导路径
- 自动布局优化
- 实时准确率评估

### 3. 可视化编辑与优化
- 图形化编辑界面
- 节点增删、关系调整
- 逻辑门修改
- 拖拽式操作

### 4. 逻辑校验与建议
- 循环依赖检测
- 孤立节点检测
- 逻辑冲突识别
- AI优化建议

### 5. 结果导出
- 支持JSON/PNG/SVG/Excel
- 包含完整分析报告
- 导出历史记录

## 操作流程

```
工业文本输入 → 知识提取层 → 故障知识提取 → 故障树生成 → 可视化呈现 → 专家修正 → 逻辑检验 → 逻辑诊断
```

### 详细步骤
1. **上传文档**：支持多文件批量上传
2. **指定顶事件**：用户指定要分析的顶事件
3. **AI提取**：大模型自动提取故障知识
4. **生成故障树**：自动构建层级结构
5. **专家优化**：可视化编辑和调整
6. **逻辑校验**：AI检查合理性
7. **导出结果**：多格式导出

## 技术亮点

### 1. AI模型选型
- **开源大模型**：Llama 3、GLM-4、ChatGLM3
- **Prompt工程**：Few-shot + Chain-of-Thought
- **结构化输出**：JSON格式约束

### 2. 知识提取算法
- **NER**：命名实体识别提取故障事件
- **RE**：关系抽取识别因果关系
- **文本分类**：区分事件类型

### 3. 故障树构建
- **DAG算法**：NetworkX有向无环图
- **拓扑排序**：确定层级结构
- **布局优化**：Graphviz层次布局

### 4. 逻辑校验
- **循环检测**：is_directed_acyclic_graph
- **孤立节点**：isolates检测
- **规则引擎**：逻辑门规则验证
