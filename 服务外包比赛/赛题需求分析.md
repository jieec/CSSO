# 第十七届服务外包大赛 A15赛题 - 需求分析

## 核心痛点
1. **故障树构建低效**：依赖人工经验，周期长、成本高
2. **知识转化困难**：分散知识难以标准化提取和复用
3. **缺乏辅助工具**：无智能辅助，迭代滞后

## 用户期望
1. **知识驱动自动生成**：基于工业知识自动提取故障关联，生成规范故障树
2. **专家辅助构建**：可视化交互，支持手动调整、逻辑补充、错误修正

## 核心功能模块

### 1. 故障树AI智能生成模块
- **工业知识提取**：从设备手册等提取故障事件、关联关系、逻辑门规则
- **自动生成故障树**：按FTA规范构建完整故障树，呈现逻辑传导路径

### 2. 专家辅助构建与优化模块
- **可视化编辑**：图形化界面，支持节点增删、关系调整、逻辑门修改
- **逻辑校验与提示**：AI自动校验合理性，给出优化建议

## 操作流程
1. **知识抽取**：上传维修手册/文本知识 → 指定顶事件 → AI提取
2. **AI生成**：生成初始故障树（结构准确率≥80%）+ 溯源依据
3. **专家优化**：可视化修改 → AI逻辑校验 → 保存最终故障树

## 技术要求
1. **核心算法**：Transformer/知识图谱/开源大模型（如Llama 3、GLM-4）
2. **数据适配**：支持文本、表格等格式，支持多数据集同时上传
3. **生成指标**：
   - 故障树结构准确率 ≥ 80%
   - 关联关系提取准确率 ≥ 85%

## 系统架构
- **后端**：FastAPI + Transformers + NetworkX + Neo4j
- **前端**：Vue 3 + Element Plus + mxGraph
- **AI模型**：开源大模型（Llama 3/GLM-4/ChatGLM3）
